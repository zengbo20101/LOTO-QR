## 一、通用前置规则（必遵守，全程生效）
### 1. 整体技术栈要求
1. 后端：**Express.js + Mongoose + MongoDB**（支持模拟数据，无需本地真实数据库也可运行）
2. 认证：**JWT 身份校验**，统一接口鉴权，所有非登录接口必须携带 Token
3. 密码：使用 `bcryptjs` 加密存储，禁止明文保存
4. 二维码：使用 `qrcode` 库生成标准二维码（Base64 格式），可微信正常扫码识别
5. CSV 导出：使用 `json2csv` 实现后端生成，适配微信小程序端导出逻辑
6. 跨域：后端开启 CORS，兼容微信小程序请求

### 2. 小程序端技术约束
1. 框架：**微信原生小程序 WXML + WXSS + 原生 JS**，不使用 uni-app、Taro 等第三方框架
2. 本地存储：会话 Token、临时数据统一使用 `wx.setStorageSync / wx.getStorageSync`
3. 网络请求：统一封装 `wx.request` 请求方法，自动携带 Token、统一错误拦截
4. 样式：基础样式简洁通用，代码分层清晰，**所有页面、组件代码独立拆分，方便单独修改**
5. 交互：加载提示、弹窗提示、权限拦截提示完善，适配手机端操作

### 3. 权限核心规则（全局生效，不可随意改动）
1. 账号注册：**仅管理员账号可执行注册操作**，普通用户无注册权限
2. LOTO 记录权限：
   - 所有登录用户：可查看全部 LOTO 记录、设备信息
   - 仅**记录创建人**：可编辑、修改、删除、解除对应 LOTO 记录
   - 非创建人：隐藏编辑/删除按钮，点击拦截并弹出无权限提示
1. 设备管理：所有登录用户可新增、查看设备（如需限制可后续单独修改）
2. 管理员账号：SARL2LOTO,密码：EHS123456

## 二、数据库模型设计（固定结构，字段可按需增删）
### 1. 用户表 Users
```
字段：
- _id：主键 ObjectId
- username：字符串，登录账号（唯一）
- password：字符串，加密后的密码
- role：字符串，角色枚举【admin / user】
- createdAt：创建时间 Date
```

### 2. 设备表 Devices（核心：编号+名称+二维码）
```
字段：
- _id：主键 ObjectId
- deviceNo：字符串，**自动生成唯一设备编号**，规则：EQP-四位年份-三位自增序号（例：EQP-2026-001）
- deviceName：字符串，手动输入设备名称
- qrContent：字符串，二维码原始内容（小程序跳转链接+设备ID参数）
- qrBase64：字符串，二维码 Base64 图片（前端直接渲染）
- createdBy：关联用户ID ObjectId（创建人）
- createdAt：创建时间 Date
```

### 3. LOTO记录表 LotoRecords
```
字段：
- _id：主键 ObjectId
- deviceId：关联设备ID ObjectId（绑定对应设备）
- lotoInfo：字符串，LOTO 详情内容
- status：字符串，状态枚举【锁定 / 已解除】
- createdBy：关联用户ID ObjectId（记录创建人）
- createdAt：创建时间 Date
- updatedAt：更新时间 Date
```

## 三、后端完整接口清单（路由+功能，可单独修改某接口）
统一基础路由前缀：`/api`，所有接口分模块编写，注释清晰。
### （一）认证模块 /api/auth
1. **登录接口** `POST /login`
   - 入参：username、password
   - 逻辑：校验账号密码，生成 JWT Token 返回，携带用户角色、用户ID
2. **注册接口** `POST /register`
   - 前置校验：必须管理员 Token 鉴权
   - 入参：username、password、role
   - 逻辑：账号去重，密码加密，新增用户

### （二）设备模块 /api/device
1. **新增设备** `POST /add`
   - 前置：JWT 鉴权
   - 入参：deviceName
   - 核心逻辑：
     1. 自动按规则生成唯一设备编号
     2. 拼接小程序扫码跳转链接（携带当前设备 _id）
     3. 生成二维码 Base64 图片
     4. 数据入库，返回完整设备信息
2. **获取设备列表** `GET /list`
   - 前置：JWT 鉴权
   - 返回：全量设备数据（编号、名称、二维码图、创建人）
3. **获取单个设备详情** `GET /detail/:id`
   - 前置：JWT 鉴权
   - 根据设备ID查询单条设备信息

### （三）LOTO记录模块 /api/loto
1. **创建LOTO记录** `POST /add`
   - 前置：JWT 鉴权
   - 入参：deviceId、lotoInfo、status
2. **获取LOTO列表** `GET /list`
   - 前置：JWT 鉴权
   - 支持筛选参数：deviceId（设备）、userId（创建人）、startTime/endTime（时间范围）
3. **修改LOTO记录** `PUT /update/:id`
   - 前置：JWT 鉴权 + **创建人权限中间件校验**
   - 入参：lotoInfo、status
4. **删除LOTO记录** `DELETE /delete/:id`
   - 前置：JWT 鉴权 + **创建人权限中间件校验**
5. **解除LOTO状态** `PUT /cancel/:id`
   - 前置：JWT 鉴权 + **创建人权限中间件校验**
   - 逻辑：仅修改 status 为「已解除」

### （四）导出模块 /api/export
1. **导出CSV** `GET /loto`
   - 前置：JWT 鉴权
   - 筛选参数：时间范围、创建人、设备ID/二维码关联设备
   - 逻辑：查询对应数据，转为标准 CSV 文本返回

### （五）公共中间件（独立文件，方便修改规则）
1. `auth.js`：JWT 鉴权中间件，校验 Token 有效性
2. `isAdmin.js`：管理员权限中间件，拦截普通用户访问注册接口
3. `isRecordOwner.js`：LOTO记录归属校验，仅创建人可操作

## 四、微信小程序页面清单（共6个页面，独立文件，可单页修改）
页面路径、功能、交互、跳转逻辑明确，**每个页面代码解耦，支持单独修改样式/逻辑**

### 页面1：登录页 `pages/login/login`
- 功能：账号密码登录、本地读取历史账号
- 交互：
  1. 登录成功：存储 Token、用户信息，跳转到首页
  2. 登录失败：弹窗提示错误原因
  3. 注册按钮：**仅管理员登录后可见**，点击跳转到注册页
- 约束：无 Token 时强制停留在登录页

### 页面2：用户注册页 `pages/register/register`
- 前置：仅管理员可进入，普通用户不可见
- 功能：输入账号、密码、选择角色，提交注册新账号
- 交互：提交成功弹窗提示，返回登录页

### 页面3：设备列表页 `pages/device/device`（核心页）
- 功能：
  1. 展示所有设备：**设备编号、设备名称、二维码图片**
  2. 新增设备按钮：点击弹出输入框，填写设备名提交
  3. 二维码操作：长按二维码，支持「保存图片到手机」
- 扫码联动：微信扫描设备二维码，自动携带 `deviceId` 参数进入 LOTO 列表页

### 页面4：LOTO列表页 `pages/loto/loto`
- 入口：手动进入 / 扫码设备二维码自动跳转（携带设备ID筛选）
- 功能：
  1. 展示 LOTO 记录：关联设备、内容、状态、创建人、时间
  2. 筛选功能：按设备、时间范围、创建人筛选
  3. 新增 LOTO 记录按钮
- 权限交互：
  - 当前登录人是记录创建人：显示【编辑】【删除】【解除】按钮
  - 非创建人：隐藏所有操作按钮

### 页面5：LOTO 新增/编辑页 `pages/loto/edit`
- 复用页面：新增、编辑共用，通过路由参数区分场景
- 入参：编辑时携带 `lotoId`，自动回显原有数据
- 权限：编辑操作仅创建人可提交，非创建人拦截

### 页面6：数据导出页 `pages/export/export`
- 功能：选择筛选条件（时间、创建人、设备），点击导出
- 小程序端导出逻辑：
  1. 调用后端接口获取 CSV 文本
  2. 支持：复制文本到剪贴板 / 分享到微信好友
  3. 提示导出结果

### 小程序公共封装（独立文件，全局修改生效）
1. `utils/request.js`：统一请求封装（Token 自动携带、错误统一处理）
2. `utils/token.js`：本地 Token 读写、清除工具方法
3. `app.wxss`：全局公共样式
